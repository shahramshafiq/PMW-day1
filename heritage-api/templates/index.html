AI-Based 3D Heritage Reconstruction API

REST API for accessing Pakistani heritage site data, point cloud exports, and 3D reconstruction pipeline simulation. Testable via curl, Postman, or direct IP hit.

Base URL: http://localhost:5000

API Endpoints

GET
/api/health
Service health check. Returns version, team, and site count.
GET
/api/sites
List all heritage sites with status, method, and preservation score.
GET
/api/sites/:id
Full details for a specific site. IDs: lahore-fort, rohtas-fort, mohenjo-daro, badshahi-mosque.
GET
/api/sites/:id/pointcloud
Sample 3D point cloud data (XYZ coordinates) for a reconstructed site.
POST
/api/reconstruct
Queue a reconstruction job. Body: {"site_id":"lahore-fort","method":"colmap","num_photos":120}
GET
/api/methods
List all 3D reconstruction methods: COLMAP, NeRF, 3DGS, MiDaS.
GET
/api/stats
Global statistics: total sites, photos processed, 3D point count, risk summary.

curl Commands

Health Check
curl http://localhost:5000/api/health
Direct IP hit: curl http://192.168.x.x:5000/api/health
List Sites
curl http://localhost:5000/api/sites
Returns all 4 heritage sites with status and preservation scores.
Site Details
curl http://localhost:5000/api/sites/lahore-fort
Full detail including era, photos, point cloud size, coverage %.
Point Cloud Data
curl http://localhost:5000/api/sites/lahore-fort/pointcloud
Returns sampled XYZ coordinates from the 3D reconstruction.
POST: Queue Reconstruction
curl -X POST http://localhost:5000/api/reconstruct \ -H "Content-Type: application/json" \ -d '{"site_id":"mohenjo-daro","method":"midas","num_photos":95}'
Queues a reconstruction pipeline job, returns job_id and estimated hours.
Postman Setup
Import URL: http://localhost:5000
No auth needed. All GET endpoints respond directly. POST /api/reconstruct requires JSON body with Content-Type header.
